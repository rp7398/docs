End‑to‑End Fraud Detection MLOps Pipeline

A modern live project report showcasing how a classical machine learning fraud detection system is productionized using industry‑grade MLOps tools and cloud‑ready infrastructure.

Introduction

This live project focuses on building a complete fraud detection pipeline that goes beyond model development and addresses orchestration, experiment tracking, artifact storage, and production parity.

Project Objectives

  • Design an end‑to‑end fraud detection ML pipeline
  • Automate training and evaluation using workflows
  • Enable experiment tracking and model versioning
  • Store ML artifacts in external object storage
  • Follow real‑world MLOps and DevOps practices

Technology Stack

PythonCore language for ML and orchestration logic
XGBoostClassical ML model for fraud detection
Apache AirflowWorkflow orchestration and scheduling
MLflowExperiment tracking and model registry
PostgreSQLMetadata and experiment database
MinIOS3‑compatible artifact storage
DockerContainerized infrastructure
WSL (Linux)Production‑parity execution on Windows

System Architecture

The system is split into two independent layers. The infrastructure layer manages storage, tracking, and messaging, while the orchestration layer controls execution and scheduling. This separation mirrors real production ML systems used in industry.

Pipeline Execution Flow

1. Data Processing
Ingestion, cleaning, and feature preparation
2. Model Training
Fraud classification using XGBoost
3. Experiment Tracking
Metrics and parameters logged to MLflow
4. Artifact Storage
Models stored in MinIO (S3‑compatible)
5. Orchestration
Automated scheduling and monitoring via Airflow

Why Linux / WSL

Most production ML systems run on Linux. Using WSL ensures consistent behavior for shell scripts, Docker containers, and permissions while working on a Windows machine. This avoids platform‑specific issues and keeps the project production‑ready.

Results & Outcomes

  • Fully automated and repeatable ML pipeline
  • Tracked and reproducible experiments
  • Persistent, cloud‑ready artifact storage
  • Clean separation of infrastructure and orchestration

Conclusion

This project demonstrates how classical machine learning can be productionized using modern MLOps principles. The design emphasizes simplicity, scalability, and real‑world applicability, making it suitable for enterprise‑grade ML systems.
